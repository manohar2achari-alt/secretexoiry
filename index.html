App Registration Monitor

Automated governance tool to detect expiring Service Principal secrets and alert the IAM team before production outages occur.

Alert Logic Strategy
To reduce alert fatigue while ensuring coverage, the system triggers alerts strictly on the following timeline:
60 Days
Early Warning
30 Days
Renewal Action
7 Days
Critical Alert
0 Days
Expiry

1. Create Automation Account

We utilize Azure Automation to run the PowerShell script serverlessly.

  • Navigate to Azure Portal > Automation Accounts > Create.
  • Resource Group: Select your Management/Identity RG.
  • Managed Identity: Ensure "System Assigned" is checked. This is critical for authentication without storing credentials.
  • Network Access: Public access (unless you require Hybrid Workers).

2. Import Modules (Critical)

Azure Automation does not include the Microsoft Graph SDK by default. The script will fail if these modules are not imported.

  • Go to your Automation Account blade > Modules.
  • Click Add a module > Browse Gallery.
  • Search for and import the following modules (Runtime: PowerShell 7.2):
    1. Microsoft.Graph.Authentication
    2. Microsoft.Graph.Applications
    3. Microsoft.Graph.Users.Actions (Required for sending mail)

3. Assign Graph Permissions

The Managed Identity (MI) requires specific Graph API permissions to Read all apps and Send email. These are "Application Permissions" and must be assigned via PowerShell.

Permissions Explained:
• Application.Read.All: Allows the script to scan all App Registrations dates.
• Mail.Send: Allows the MI to send email via Graph API as the Shared Mailbox.
PowerShell (Run locally as Admin) 
# 1. Connect to Graph (Requires Global Admin or Privileged Role Admin)
Connect-MgGraph -Scopes "AppRoleAssignment.ReadWrite.All","Application.Read.All"

# 2. Configuration
$IdentityName = "aa-secret-monitor-prod" # Name of your Automation Account
$MI = Get-MgServicePrincipal -Filter "displayName eq '$IdentityName'"
$GraphApp = Get-MgServicePrincipal -Filter "appId eq '00000003-0000-0000-c000-000000000000'"

# 3. Roles to Assign
$Roles = @("Application.Read.All", "Mail.Send")

# 4. Assign Roles
foreach ($RoleName in $Roles) {
    $RoleID = $GraphApp.AppRoles | Where-Object {$_.Value -eq $RoleName}
    New-MgServicePrincipalAppRoleAssignedTo `
        -ServicePrincipalId $MI.Id `
        -PrincipalId $MI.Id `
        -ResourceId $GraphApp.Id `
        -AppRoleId $RoleID.Id
    Write-Host "assigned $RoleName" -ForegroundColor Green
}

4. Mailbox Configuration

We send the email via the Microsoft Graph API, acting "as" a shared mailbox.

  • Sender Address: Ensure you have a valid shared mailbox (e.g., alerts@domain.com).
  • No "Send As" Delegation needed: Because we use the Mail.Send Application Permission, the Managed Identity can send mail as any user in the tenant. We restrict the "From" address in the script logic below.

5. The Runbook Script

Create a new Runbook (PowerShell 7.2). This script aggregates all alerts into one summary email.

Runbook Source Code 
# --- CONFIGURATION VARIABLES ---
# Only trigger alerts if expiry is exactly X days away
$NotifyDays = @(60, 30, 7) 

$SenderEmail = "alerts@yourdomain.com"      # The Shared Mailbox
$RecipientEmail = "iam-team@yourdomain.com" # The DL receiving alerts

# --- AUTHENTICATION ---
try {
    Connect-AzAccount -Identity -ErrorAction Stop
    $token = Get-AzAccessToken -ResourceUrl "https://graph.microsoft.com"
    Connect-MgGraph -AccessToken $token.Token -ErrorAction Stop
} catch {
    Write-Error "Failed to authenticate Managed Identity. Check Module imports."
    exit
}

# --- SCANNING LOGIC ---
$ExpiringApps = @()
$Today = Get-Date
$Apps = Get-MgApplication -All -Property DisplayName, AppId, PasswordCredentials

Write-Output "Scanning $($Apps.Count) applications..."

foreach ($App in $Apps) {
    foreach ($Cred in $App.PasswordCredentials) {
        $Expiry = Get-Date $Cred.EndDateTime
        $DaysLeft = ($Expiry - $Today).Days

        # CHECK: Is today exactly 60, 30, or 7 days before expiry?
        if ($NotifyDays -contains $DaysLeft) {
            $ExpiringApps += [PSCustomObject]@{
                Name = $App.DisplayName
                AppId = $App.AppId
                DaysLeft = $DaysLeft
                ExpiryDate = $Expiry.ToString("yyyy-MM-dd")
            }
        }
    }
}

# --- NOTIFICATION ---
if ($ExpiringApps.Count -gt 0) {
    Write-Output "Found $($ExpiringApps.Count) secrets expiring in notification window."
    
    # Build HTML Table
    $TableRows = ""
    foreach ($Row in $ExpiringApps) {
        $Color = if($Row.DaysLeft -eq 7){"#ffcccb"} else {"#ffffff"}
        $TableRows += "<tr bgcolor='$Color'><td>$($Row.Name)</td><td><b>$($Row.DaysLeft)</b></td><td>$($Row.ExpiryDate)</td></tr>"
    }

    $EmailBody = "
    <h3>⚠️ App Secrets Expiring</h3>
    <p>The following applications have secrets expiring in 60, 30, or 7 days.</p>
    <table border='1' cellpadding='8' style='border-collapse:collapse; width:100%; font-family:sans-serif;'>
        <tr bgcolor='#f2f2f2' align='left'><th>App Name</th><th>Days Left</th><th>Expiry Date</th></tr>
        $TableRows
    </table>"

    $Params = @{
        Message = @{
            Subject = "Action Required: $($ExpiringApps.Count) Apps Expiring"
            Body = @{ ContentType = "HTML"; Content = $EmailBody }
            ToRecipients = @( @{ EmailAddress = @{ Address = $RecipientEmail } } )
        }
        SaveToSentItems = "false"
    }
    
    Send-MgUserMail -UserId $SenderEmail -BodyParameter $Params
    Write-Output "Email sent successfully."
} else {
    Write-Output "No secrets match the 60/30/7 day trigger window today."
}

6. Scheduling

To ensure the script catches the specific day count (e.g., exactly 30 days remaining), it must run every day.

  • In the Automation Account, select Schedules under "Shared Resources".
  • Create a schedule:
    • Name: Daily-8AM-Check
    • Recurrence: Recurring
    • Recur every: 1 Day
  • Go back to your Runbook, click Link to schedule, and select the schedule you just made.
