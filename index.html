App Registration Secret Monitor

A "set and forget" solution to prevent production outages caused by expired Service Principal secrets. This tool runs daily in Azure Automation, scanning for credentials expiring within 60, 30, and 7 days.

Prerequisites

Role Required: You need Global Administrator or Privileged Role Administrator to assign Graph permissions to a Managed Identity.

1. Create Automation Account

Create a new Azure Automation resource. Ensure System Assigned Managed Identity is enabled during creation.

Once created, go to Identity in the left blade and copy the Object (principal) ID.

2. Assign Graph Permissions

The Managed Identity needs permission to read all applications and send emails. These permissions cannot be added via the Azure Portal GUI; they must be added via PowerShell.

PowerShell (Local Machine) 
# 1. Connect as an Admin
Connect-MgGraph -Scopes "AppRoleAssignment.ReadWrite.All","Application.Read.All"

# 2. Define Variables
$ManagedIdentityName = "aa-secret-monitor-prod" # Your Automation Account Name
$MI = Get-MgServicePrincipal -Filter "displayName eq '$ManagedIdentityName'"
$GraphApp = Get-MgServicePrincipal -Filter "appId eq '00000003-0000-0000-c000-000000000000'"

# 3. Define Permissions Needed
$Perms = @("Application.Read.All", "Mail.Send")

# 4. Loop and Assign
foreach ($Perm in $Perms) {
    $Role = $GraphApp.AppRoles | Where-Object {$_.Value -eq $Perm}
    New-MgServicePrincipalAppRoleAssignedTo `
        -ServicePrincipalId $MI.Id `
        -PrincipalId $MI.Id `
        -ResourceId $GraphApp.Id `
        -AppRoleId $Role.Id
    Write-Host "Assigned $Perm" -ForegroundColor Green
}

3. Configure Shared Mailbox

For the automation to send emails "from" a specific address, the Managed Identity must be granted Send As permissions on that mailbox.

You can do this via the Microsoft 365 Admin Center (Exchange > Mailbox > Delegation) or via Exchange Online PowerShell.

4. The Runbook Script

Create a new Runbook (PowerShell 7.2). Paste the script below. This improved version aggregates all expiring apps into a single HTML table.

Runbook Source Code 
# --- CONFIGURATION ---
$NotifyDays = @(60, 30, 7)
$SenderEmail = "alerts@yourdomain.com"
$RecipientEmail = "iam-team@yourdomain.com"

# --- AUTHENTICATION ---
Connect-AzAccount -Identity
$token = Get-AzAccessToken -ResourceUrl "https://graph.microsoft.com"
Connect-MgGraph -AccessToken $token.Token

$ExpiringApps = @()
$Today = Get-Date
$Apps = Get-MgApplication -All -Property DisplayName, AppId, PasswordCredentials

# --- LOGIC ---
foreach ($App in $Apps) {
    foreach ($Cred in $App.PasswordCredentials) {
        $Expiry = Get-Date $Cred.EndDateTime
        $DaysLeft = ($Expiry - $Today).Days

        if ($NotifyDays -contains $DaysLeft) {
            $ExpiringApps += [PSCustomObject]@{
                Name = $App.DisplayName
                AppId = $App.AppId
                DaysLeft = $DaysLeft
                ExpiryDate = $Expiry.ToString("yyyy-MM-dd")
            }
        }
    }
}

# --- NOTIFICATION ---
if ($ExpiringApps.Count -gt 0) {
    $TableRows = ""
    foreach ($Row in $ExpiringApps) {
        $TableRows += "<tr><td>$($Row.Name)</td><td>$($Row.DaysLeft)</td><td>$($Row.ExpiryDate)</td></tr>"
    }

    $EmailBody = "
    <h3>‚ö†Ô∏è App Secrets Expiring</h3>
    <table border='1' cellpadding='5' style='border-collapse:collapse;'>
        <tr bgcolor='#f2f2f2'><th>App Name</th><th>Days Left</th><th>Expiry Date</th></tr>
        $TableRows
    </table>"

    $Params = @{
        Message = @{
            Subject = "Action Required: $($ExpiringApps.Count) Apps Expiring"
            Body = @{ ContentType = "HTML"; Content = $EmailBody }
            ToRecipients = @( @{ EmailAddress = @{ Address = $RecipientEmail } } )
        }
        SaveToSentItems = "false"
    }
    
    Send-MgUserMail -UserId $SenderEmail -BodyParameter $Params
    Write-Output "Alert sent for $($ExpiringApps.Count) apps."
} else {
    Write-Output "No secrets expiring in target window."
}

5. Schedule

In the Automation Account, go to Schedules > Add a schedule.

  • Set recurrence to Daily.
  • Link the schedule to your Runbook.

We recommend running this at 8:00 AM in your local time zone so the alerts are at the top of the inbox.
